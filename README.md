# PyFyve

A fully offline, free Python tutoring application that teaches through Socratic hints — not answers.

---

## What It Does

PyFyve gives beginners a structured Python curriculum with a sandboxed code editor. When a student makes a mistake, a locally-running fine-tuned AI model diagnoses the error and responds with exactly three sentences: what went wrong, which Python rule was violated, and a question that guides the student toward the fix — without ever stating the answer.

No internet required at runtime. No API costs. Runs entirely on the student's machine.

---

## Background

PyFyve started from a single question: *what if the tutor refused to give you the answer?*

Most coding tools — AI assistants, Stack Overflow, even traditional tutors — default to showing the solution. That removes the cognitive struggle that actually builds understanding. PyFyve is built around the Socratic method: diagnose the specific mistake, state the rule, ask a question. The student has to complete the final step themselves.

The AI model is not a general-purpose LLM. It is a fine-tuned Qwen3 4B model trained specifically on this narrow task — 555 curated (error, hint) pairs generated by a 30B teacher model, then distilled down. It knows how to give Socratic Python hints. It does not know how to do anything else, which is by design.

## How It Works

```
Student writes code in Notepad++ (bundled)
        ↓
Code is sandboxed and executed via exec()
        ↓
Result routed to validator (checks variables, output, types, collections)
        ↓
On failure: error classified → AI model generates 3-line Socratic hint
        ↓
Hint streamed to terminal with typewriter effect
```

**Hint structure (always exactly 3 lines):**
1. **Diagnosis** — names the specific variable, expression, or construct that caused the error
2. **Rule** — states the Python rule that was violated
3. **Directive** — starts with *"Think about..."* or *"Consider..."* and guides without giving the answer

## Lesson Format

Each lesson is a JSON file in `lessons/`:

```json
{
  "id": 1,
  "topic": "Variables & Assignment",
  "text": "...",
  "common_errors": "Writing `100 = score`. The variable name MUST be on the left.",
  "task": "Create variables named 'health' and 'stamina' and assign them 100 and 90.",
  "validation": [
    {"type": "variable_check", "var_info": {"health": 100, "stamina": 90}}
  ]
}
```

**Supported validation types:**
- `variable_check` — checks variable names and values, with type mismatch detection
- `output_check` — checks printed output matches expected (whitespace-stripped)
- `type_check` — checks variable type only, not value
- `source_check` — checks if required syntax appears in user code (e.g. forces use of a loop)
- `collection_check` — checks lists/dicts for size, type, and contents

## Setup

> **Windows only** for now. Linux/Mac support is planned.

**Prerequisites:**
- Python 3.10+
- [Ollama](https://ollama.com) installed

**Steps:**

1. Clone the repo
2. Run `start.bat` — this sets up the virtual environment and installs dependencies
3. On first run, `setup.py` checks that the `fyve-ai` model is available in Ollama
4. To install the model: `ollama create fyve-ai -f Modelfile` (Modelfile and GGUF download link in releases)

## Limitations

**These are real, not disclaimers:**

- **Windows only** — the editor integration uses Notepad++. Linux/Mac use `nano` as a fallback but this is untested.
- **Ollama dependency** — the student needs Ollama installed. Migration to `llama-cpp-python` (no external dependency) is planned.
- **No logical error hints yet** — the AI fires on Python exceptions. When code runs correctly but produces the wrong result, validation fails silently with no AI guidance. This is a known gap and the next major AI feature.
- **No infinite loop protection** — a `while True: pass` will hang the app. A process watchdog is on the roadmap.
- **Placeholder curriculum** — the lesson content included is minimal. Writing the full curriculum is ongoing work.
- **Model trained on beginner/intermediate errors only** — the fine-tuned model knows common syntax and runtime errors well. Unusual or advanced errors may produce weaker hints.
- **Few-shot prompting required at inference** — the model's Socratic behaviour is anchored by few-shot examples in the prompt at runtime. Removing them degrades output quality.

## The AI Model

The `fyve-ai` model is a fine-tuned Qwen3 4B, not a general assistant. It was trained on 555 curated examples of (broken code, Socratic hint) pairs across beginner and intermediate Python error types. Training was done with QLoRA via Unsloth on a free Colab T4 GPU.

The model outputs a JSON object with two keys: `reasoning` (internal diagnosis) and `hint` (the 3-sentence student-facing response). Output is validated structurally and semantically before being shown to the student. If validation fails, the hint is discarded and the AI retries.

The teacher model used for dataset generation (Qwen3 30B-A3B) is not shipped with the app. It runs locally on the developer's machine for pipeline work only.
